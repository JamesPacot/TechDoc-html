Introduction

HTML (Hypertext Markup Language) is the standard language used to structure and display content on the web. It defines the content and structure of a webpage using a system of elements or "tags" that provide the framework for text, images, multimedia, links, and more. HTML is the backbone of nearly every website and is used alongside other technologies like CSS (Cascading Style Sheets) and JavaScript to create modern web pages and applications.

An html is defined like this inside of an html file


                <!DOCTYPE html>
                <html lang="en">
                <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Document
                </head>
                <body>
                    
                </body>
                </html>

Purpose:

  • To provide structure and meaning to the content of a webpage.
  • To create the foundation upon which all other web technologies (CSS, JavaScript, etc.) interact.
  • HTML is the building block of the web, used to represent documents in a readable format.

Common Use Cases:

  • Building websites, blogs, and web applications.
  • Formatting content for display on browsers.
  • Creating interactive forms and user interfaces.

Tags

HTML tags are the building blocks of a webpage's structure. They define different parts of a web page and provide the necessary information for browsers to render and display content correctly. Tags usually come in pairs, with an opening tag and a closing tag, although some tags are self-closing.

Key html tags: 


            <p></p>, <a href=""></a>, <ul></ul>
        

            <ol></ol>, <img src="" alt="">, <div></div>
        

            <span></span>, <form action=""></form>

Purpose:

  • To structure and organize content for web browsers.
  • To add semantic meaning to content, such as defining headings, paragraphs, and links.

Forms

Forms in HTML are used to collect input from users and send the data to a server for processing. HTML forms consist of various input elements, such as text boxes, radio buttons, checkboxes, and submit buttons. The data entered into these form fields can be sent to a server for processing, often using methods like GET or POST.

Common Form Elements: 


            <input type="text">, <label for=""></label>, <select name="" id=""></select>
        

            <textarea name="" id=""></textarea>, <button></button>

Purpose:

  • To gather data from users in a structured and accessible manner.
  • To create interactive forms for tasks like user registration, login, surveys, and feedback collection.

Buttons

Buttons are HTML elements that users can click to perform specific actions. These actions could include submitting a form, resetting input fields, or triggering JavaScript functionality. Buttons are defined using the <button> tag or the <input type="button"> element, each with different use cases.

Types of buttons: 


        <button></button>, <input type="submit">, <input type="reset">, <input type="button">

Purpose:

  • To provide interactivity on a webpage.
  • To initiate actions such as form submissions, navigating to another page, or executing JavaScript events.

Document Object Model (DOM)

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of nodes, where each node is an object representing part of the page, such as an element, attribute, or piece of text. JavaScript can manipulate the DOM to dynamically update the content, structure, and styling of a webpage.

Key Concepts of DOM:

  • DOM Tree Structure: The DOM represents the page in a hierarchical structure, where the <html> element is the root, and all other elements are child nodes.
  • Node Types: There are several types of nodes, including element nodes (e.g., <div>, <p>), text nodes (e.g., the text within a paragraph), and attribute nodes (e.g., the class attribute).
  • DOM Manipulation: JavaScript can interact with and modify the DOM to change content, add new elements, or respond to user actions.

Common DOM Methods:

  • getElementById(): Selects an element by its ID.
  • getElementsByClassName(): Selects elements by their class name.
  • createElement(): Creates a new HTML element.
  • innerHTML: Sets or retrieves the HTML content inside an element.

Purpose:

  • To enable dynamic content updates and interactive features on a webpage.
  • To provide developers with a way to modify the content and structure of the page in response to user interactions.
